# Quick Task 260629-1fb: Standardize auth on /api/v1/auth

## Goal

Eliminate the bare-`/auth/*` inconsistency (latent prod-routing bug) by moving identity's
customer-auth endpoints under `/api/v1/auth`, aligning every client call path to the
canonical `/api/v1/*` contract, and removing the temporary gateway stop-gap.

## Audit result (full client↔server diff)

- **Server**: only 3 identity controllers use bare `/auth` — `RegistrationController`
  (`/register`), `SessionController` (`/me /login /refresh /logout`), `PasswordResetController`
  (`/forgot /reset`). Everything else (incl. `AdminLoginController` = `/api/v1/auth/admin`)
  is already `/api/v1/*`.
- **admin-web**: every call already uses `/api/v1/*` — **no changes needed**.
- **Flutter**: only auth is off — 4 sites call bare `/auth/*`.
- **SecurityConfig**: permitAll matchers list `/auth/{register,login,refresh,forgot,reset}` —
  MUST move too or those endpoints 401 after the controller move.

## Safe-replacement rule

`/api/v1/auth/admin` contains the substring `/auth/admin`, so replacements are anchored to
the literal delimiter to avoid corrupting admin paths:
- `"/auth/` → `"/api/v1/auth/`  | `"/auth"` → `"/api/v1/auth"`  (Java code/tests)
- `'/auth/` → `'/api/v1/auth/`  (Dart)
- ` /auth/` → ` /api/v1/auth/`  (javadoc/comments)
None of these match `"/api/v1/auth/admin…` / ` /api/v1/auth/admin…`.
