<objective>
Implement registration (IDEN-01), async NIDA verification returning PENDING immediately (IDEN-02), and graceful degrade + background retry (IDEN-08). This plan owns the registration entry point, the NIDA service (interface + configurable stub + real impl), the async orchestrator, and the scheduled retry job. The actual VERIFIED state transition (status flip + sender ID + outbox) is owned by 02-06 behind the `VerificationFinalizer` seam this plan defines.

Purpose: Keep registration fast and resilient — async-on-bounded-executor + circuit breaker. Defining the finalizer as an interface here lets the orchestrator compile and run in Wave 2 while 02-06 supplies the transactional implementation in Wave 3.
Output: Registration endpoint + service, NIDA interface with stub (configurable outcomes) and real impls, VerificationOrchestrator (async, calls the finalizer seam on success), VerificationFinalizer interface, scheduled retry job.
</objective>
