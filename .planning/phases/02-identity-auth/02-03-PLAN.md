<objective>
Implement registration (IDEN-01), async NIDA verification returning PENDING immediately (IDEN-02), graceful degrade + background retry (IDEN-08), the verification→VERIFIED state machine that grants 50 credits via transactional outbox (IDEN-03) and assigns a default numeric sender ID (SNDR-01).

Output: Registration endpoint + service, NIDA interface with stub (configurable outcomes) and real impls, verification orchestrator (single-TX status flip + sender ID + outbox), scheduled retry job, scheduled outbox relay, V2/V3 migrations.

| Boundary | Description |
|----------|-------------|
| client → /auth/register | untrusted email/phone/nin/password; @Valid + dup checks |
| identity → NIDA (external) | slow/unreliable government API; bounded executor + circuit breaker |
| identity outbox → RabbitMQ → Phase 3 wallet | async credit-grant boundary; at-least-once + eventId dedupe |
