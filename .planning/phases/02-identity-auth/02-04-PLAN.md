<objective>
Implement login (IDEN-04), persistent multi-device sessions with rotating refresh tokens (IDEN-05), logout revoke-current (IDEN-06), and brute-force lockout (D-12).

Purpose: Opaque refresh tokens in Redis are the only design that supports rotation (D-08), revoke-current (D-07), and revoke-all (D-09) — a stateless JWT cannot be revoked server-side. This plan owns the session lifecycle; the revoke-all entry point it exposes is consumed by password reset (Plan 05).
Output: Login/refresh/logout endpoints, LoginService, RefreshTokenService (rotation + reuse detection + revoke-current + revoke-all), LoginAttemptService (Redis lockout).
</objective>
