<objective>
Build the load-bearing JWT issuance + validation core and the User aggregate that every other Phase 2 plan and all 8 downstream modules depend on.

Purpose: Asymmetric RSA signing (identity signs, shared-security validates) is the mechanism that lets the verification_status claim (D-02) be trusted everywhere with NO runtime callback to identity. This plan defines the contracts; later plans build registration/login/reset against them.
Output: JwtConfig (encoder + decoder beans), SecurityConfig, RedisConfig, AsyncConfig (bounded NIDA executor), User entity + VerificationStatus + repository + UserDetailsService, V1 users migration, and the shared-security validation beans + claim accessors.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → identity REST | untrusted register/login input; STATELESS, CSRF-off JWT chain |
| identity (private key) → all modules (public key) | only identity can sign; modules can only verify |
| identity service → Postgres / Redis | credential hashes at rest; ephemeral session state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Spoofing | JWT forgery by a downstream module | mitigate | RSA asymmetric — modules hold public key only (JwtConfig); forged token test proves rejection |
| T-02-01 | Info Disclosure | password storage | mitigate | BCrypt via DelegatingPasswordEncoder; passwordHash never serialized in DTOs |
| T-02-V6 | Info Disclosure | RSA private key | mitigate | loaded from config/Secret; never logged (V6/V7); only public key in shared-security |
| T-02-DoS | DoS | NIDA async executor | mitigate | bounded ThreadPoolTaskExecutor (Pitfall 2), not unbounded virtual-thread executor |
| T-02-AC | Elevation | access control | mitigate | STATELESS chain, permitAll only on auth endpoints, anyRequest authenticated |
| T-02-SC | Tampering | dependencies | accept | all BOM-managed per RESEARCH Package Legitimacy Audit |
</threat_model>
