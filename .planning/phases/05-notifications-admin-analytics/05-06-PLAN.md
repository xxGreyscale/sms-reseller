## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AMQP exchanges → notification consumers | Events from other services (trusted broker, but may be replayed) |
| client JWT → /api/v1/notifications | Per-user feed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-14 | Tampering | duplicate event double-creates notification | mitigate | processed_events.tryInsert idempotency gate before every write |
| T-05-15 | Denial of Service | exchange re-declaration PRECONDITION_FAILED | mitigate | ignoreDeclarationExceptions="true" on every @Exchange (passive bind) |
| T-05-16 | Information Disclosure | feed returns another user's notifications | mitigate | Feed scoped to auth.getToken().getSubject() |
</threat_model>
