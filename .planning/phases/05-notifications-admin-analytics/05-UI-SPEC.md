# Phase 5 — UI Design Contract (admin-web)

> Visual and interaction contract for the operator-facing admin-web panel.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope:** admin-web Next.js 14 application only. Customer-facing Flutter UI is Phase 6.
> Six operator screens: sender-ID approval queue, user search, user ledger inspection,
> manual refund, bundle catalog management, audit-log viewer.

---

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, table cell inline padding |
| sm | 8px | Compact row spacing, button icon gap |
| md | 16px | Default card padding, form field gap, nav item padding |
| lg | 24px | Section heading margin, card-to-card gap |
| xl | 32px | Page content area top padding |
| 2xl | 48px | Major section breaks between page regions |
| 3xl | 64px | Not used — admin-web has no marketing/hero sections |

Exceptions:
- Table row height: 44px minimum (accessibility touch target floor, WCAG 2.5.5)
- Sidebar width: 240px fixed (does not follow token scale — layout constraint)
- Top nav height: 56px fixed

## Typography

Four sizes, two weights. Admin-web is data-dense — avoid large display text.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (Regular) | 1.5 | Table cell content, form input text, description paragraphs |
| Label | 12px | 500 (Medium) | 1.4 | Table column headers, form field labels, badge text, nav sub-labels |
| Heading | 18px | 600 (Semibold) | 1.3 | Page titles (h1), section titles (h2) |
| Display | 24px | 600 (Semibold) | 1.2 | Stat callouts (e.g. total pending sender-IDs count on queue page) |

Weights declared: Regular (400) and Semibold (600). Medium (500) is used only for labels and is provided by Inter at no extra weight load.

Font stack: `font-family: 'Inter', ui-sans-serif, system-ui, -apple-system, sans-serif`

## Color

Zinc base palette (shadcn default). All values are Tailwind zinc equivalents.

| Role | Tailwind Token | Approximate Hex | Usage |
|------|----------------|-----------------|-------|
| Dominant (60%) | zinc-50 | #fafafa | Page background, main content area |
| Secondary (30%) | zinc-100 / zinc-900 (dark sidebar) | #f4f4f5 / #18181b | Sidebar background, table row alternates, card surfaces |
| Accent (10%) | zinc-900 (light) / zinc-50 (text on dark) | #18181b | Primary action buttons, active sidebar nav item, approval badge |
| Destructive | red-600 | #dc2626 | Reject sender-ID button, delete/refund confirm dialogs only |
| Success semantic | green-600 | #16a34a | Approved badge, success toast, refund-confirmed state |
| Warning semantic | amber-600 | #d97706 | Pending badge, low-credit indicator in ledger |

Accent reserved for ONLY:
1. Primary action button fill (e.g. "Approve", "Save", "Search")
2. Active sidebar navigation item background
3. "Pending" count badge on the sender-ID queue nav item

Destructive color is used ONLY on:
1. "Reject" button in sender-ID approval row
2. Confirmation dialog submit button for irreversible actions (refund confirm, bundle delete)

Do NOT use accent color on table row hovers, informational icons, or secondary navigation. Use zinc-200 for hover states instead.

## Screen-by-Screen Interaction Contract

### Login (`/login`)

| Element | Spec |
|---------|------|
| Layout | Centered card, 400px wide, vertically centered on viewport |
| Fields | Email (type=email, autocomplete=email) + Password (type=password, autocomplete=current-password) |
| Submit | Button full-width: "Sign in" |
| Error state | Inline below form: "Invalid email or password. Check your credentials and try again." (non-field, red-600 text, 14px) |
| Loading state | Button shows spinner + "Signing in…" — disabled during Server Action |
| Success | Redirect to `/(admin)/sender-ids` via `redirect()` in Server Action |
| Auth wall | If cookie already set, middleware redirects away from `/login` to `/sender-ids` |

### Sender-ID Approval Queue (`/(admin)/sender-ids`) — ADMN-04

| Element | Spec |
|---------|------|
| Stat callout | Card row at top: "Pending Review: N" (24px semibold, zinc-900) |
| Filter | Select dropdown: All / Pending / Approved / Rejected — default: Pending |
| Table columns | Requested By (user email), Sender ID (alphanumeric, monospace), Submitted (relative date), Status (Badge), Actions |
| Row actions | Two buttons per pending row: "Approve" (accent fill) + "Reject" (destructive outline) |
| Reject flow | Opens Dialog: heading "Reject Sender ID", Textarea label "Reason for rejection (required)", 500-char max, Submit "Confirm Rejection" (destructive fill), Cancel |
| Approve flow | Inline confirmation: button changes to "Approving…" spinner, then success Toast "Sender ID approved" |
| Empty state | When filter=Pending + no items: Icon (check-circle, lucide), heading "Queue is clear", body "No sender ID requests are pending review." |
| Page refresh | Server Component re-fetch after mutation — use `revalidatePath('/sender-ids')` in Server Action |

### User Search (`/(admin)/users`) — ADMN-02

| Element | Spec |
|---------|------|
| Search | Input + Button row at top: placeholder "Search by email or phone…", Button "Search" |
| Table columns | Full Name, Email, Phone, Status (Badge: VERIFIED / PENDING / SUSPENDED), Registered (date), Actions |
| Row action | "View Ledger" link-button — navigates to `/ledger/[userId]` |
| Empty state (no search) | Heading "Search for a user", body "Enter an email address or phone number above." (no icon) |
| Empty state (no results) | Heading "No users found", body "No accounts match "{query}". Check the spelling or try a phone number." |
| Pagination | Bottom: shadcn Pagination, 20 rows per page |

### Ledger Inspection (`/(admin)/ledger/[userId]`) — ADMN-03

| Element | Spec |
|---------|------|
| Back navigation | "← Back to users" link at top left (14px, zinc-600) |
| User summary | Card: name, email, current balance (18px semibold), verification status Badge |
| Table columns | Date, Type (credit/debit Badge), Description, Delta (green for credit, red for debit), Balance After |
| ScrollArea | Wrap table — max-height: calc(100vh - 280px) |
| Empty state | Heading "No transactions", body "This user has no credit history yet." |

### Manual Refund (`/(admin)/refunds`) — ADMN-05

| Element | Spec |
|---------|------|
| Form | User ID or email Input, Amount (TZS) Input (number, min=1), Reason Textarea (required, 10–500 chars), Submit Button "Issue Refund" |
| Confirmation Dialog | Before submit: "Refund {amount} TZS to {userEmail}? This cannot be undone." — "Confirm Refund" (destructive fill) + "Cancel" |
| Success | Toast "Refund issued successfully. {amount} TZS credited to {userEmail}." — form resets |
| Error | Inline below form: error message from backend (e.g. "User not found" / "Amount exceeds policy limit") |

### Bundle Catalog (`/(admin)/bundles`) — ADMN-07

| Element | Spec |
|---------|------|
| Page action | Top-right Button "Add Bundle" — opens Dialog |
| Table columns | Bundle Name, SMS Count (number), Price TZS (formatted: "12,000 TZS"), Active (toggle Badge), Actions |
| Row actions | "Edit" (outline button) + "Delete" (destructive outline, icon only with tooltip) |
| Edit/Add Dialog | Fields: Name (Input), SMS Count (number, min=1), Price TZS (number, min=1), Active (Checkbox). Submit "Save Bundle" (accent fill) |
| Delete confirmation | Dialog: "Delete {bundleName}? Users will no longer be able to purchase this bundle." — "Delete" (destructive fill) + "Cancel" |
| Price display | Always formatted with comma separator and "TZS" suffix — never raw integer |

### Audit Log (`/(admin)/audit`) — ADMN-06

| Element | Spec |
|---------|------|
| Filters | Date range (two date Inputs: From / To) + Actor Select (All / Admin mutations / System events) |
| Table columns | Timestamp (full ISO, no relative), Actor (admin email or "system"), Action (monospace label e.g. `SENDER_ID_APPROVED`), Target (userId or entityId), Details (collapsed truncated text, expand on click) |
| Details expand | Inline row expansion (not a Dialog) — show raw JSON payload in `<pre>` block, zinc-100 bg, 12px monospace |
| ScrollArea | Wrap table — same max-height pattern as ledger |
| Empty state | Heading "No audit events", body "No events match the selected filters. Widen the date range or clear filters." |
| Pagination | 50 rows per page (higher density — audit log is read-heavy) |

## State Patterns (all screens)

| State | Pattern |
|-------|---------|
| Loading | shadcn `Skeleton` rows — match column count of the target table. Show 5 skeleton rows. |
| Error (data fetch) | Full-width alert below nav: "Failed to load data. Refresh the page or contact engineering." (amber-600 border, zinc-900 text, no icon) |
| Mutation success | `sonner` Toast — green-600 — top-right — 4-second auto-dismiss |
| Mutation error | `sonner` Toast — red-600 — top-right — stays until dismissed |
| Empty (no data) | Centered in table area: heading (18px semibold) + body (14px zinc-600) — no illustration |
| Confirmation required | shadcn `Dialog` — always for destructive or irreversible actions (Reject with reason, Refund confirm, Bundle delete) |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| WCAG 2.1 AA contrast | zinc-900 on zinc-50 = 16:1 (exceeds 4.5:1 body requirement). red-600 on white = 5.9:1. green-600 on white = 4.6:1 (passes). amber-600 on white = 3.1:1 — **use amber-700 (#b45309) for text, amber-100 for bg to maintain contrast** |
| Touch targets | 44px minimum row height and button height |
| Keyboard navigation | All shadcn components are Radix-based — full keyboard navigation built in. Tab order: sidebar nav → page content → table actions |
| Focus ring | shadcn default: 2px ring-zinc-950 offset-2 — do not remove |
| Dialog focus trap | shadcn Dialog uses Radix Dialog — focus trapped automatically |
| Screen reader | All icon-only buttons must have `aria-label`. Table columns must have `scope="col"` |
| Form validation | Use `aria-describedby` on inputs linked to error messages |
