# Phase 6 — UI Design Contract: Flutter Customer Mobile App

> Visual and interaction contract for the Flutter customer app. Generated by gsd-ui-researcher.
> Consumed by gsd-planner, gsd-executor, gsd-ui-auditor, and gsd-ui-checker.

This is a **native mobile app** (Flutter + Material 3), not a web app. shadcn/ui and Tailwind do not
apply. Design tokens map to Flutter `ThemeData` and `TextTheme`. All spacing values are logical pixels
(dp), not CSS pixels. All copy must be expressible as l10n ARB keys in both English (`app_en.arb`)
and Swahili (`app_sw.arb`) — the ARB key is included alongside every copy entry.

## Spacing Scale

All values are logical pixels (dp). Flutter `EdgeInsets` and `SizedBox` use these tokens directly.

| Token | Value | Flutter Usage |
|-------|-------|---------------|
| xs | 4 dp | Icon-to-label gap; `ListTile` leading padding adjustment |
| sm | 8 dp | Compact row spacing; chip padding; `Card` content padding (horizontal) |
| md | 16 dp | Default screen horizontal padding; `ListTile` content padding; form field spacing |
| lg | 24 dp | Section header bottom margin; bottom navigation safe area inset |

| 2xl | 48 dp | Splash logo vertical center offset; section break on dashboard |
| 3xl | 64 dp | App bar height on splash screen |

**Exceptions:**
- Touch targets: minimum 48 dp × 48 dp on all interactive elements (Material `kMinInteractiveDimension = 48.0`). Icon-only buttons (`IconButton`) use `iconSize: 24` with 48 dp hit area via `minimumSize`.

- FAB (Quick-send shortcut on dashboard): standard Material `FloatingActionButton` — 56 dp diameter. Extended FAB (`FloatingActionButtonExtended`) for the explicit "Send SMS" label.
- Countdown timer display: 72 dp font size (display-sized, not in the typography table — one-off for the 2-minute STK countdown clock face only).

## Typography

Flutter `TextTheme` roles. All sizes in logical pixels (sp, which equals dp on 1× density and scales with accessibility font size). Do NOT use hardcoded `fontSize` outside the theme — always reference `Theme.of(context).textTheme.*`.

| Role | TextTheme Key | Size | Weight | Line Height | Usage |
|------|--------------|------|--------|-------------|-------|
| Body | `bodyMedium` | 16 sp | 400 (normal) | 1.5 | List item text; form input text; notification body; message composer body |
| Label | `labelMedium` | 14 sp | 500 (medium) | 1.4 | Form field labels; chip labels; character counter text; badge counts |
| Heading | `titleMedium` | 20 sp | 600 (semiBold) | 1.3 | Screen titles; card section headers; bundle names; campaign names |
| Display | `headlineLarge` | 28 sp | 700 (bold) | 1.2 | SMS balance number on dashboard; STK countdown digits |

**Secondary type role (supporting, not a 5th role):**
- `bodySmall` at 12 sp / weight 400 / line-height 1.4: timestamp text, stale-cache indicator, per-message status chips — used sparingly, never for primary content.

**Rules:**
- Maximum 4 visible type sizes on any single screen.
- Swahili text tends 10–15% longer than English — all text containers must be flexible (no fixed-width `Text` without overflow handling). Use `softWrap: true` and `overflow: TextOverflow.ellipsis` on single-line labels; `maxLines: null` on body copy.
- TZS amounts: display as `"TZS 5,000"` (integer, comma-separated thousands). Use `NumberFormat('#,###', 'sw_TZ')` from the `intl` package. Never display `.00` decimals — TZS has no useful sub-unit for end-users.

## Color

Material 3 ColorScheme. The app targets a **trust-first, utility aesthetic** — clarity over decoration. Use `ColorScheme.fromSeed(seedColor: ...)` with `brightness: Brightness.light` as the base; dark mode is deferred (not in MVP scope).

| Role | Value | Flutter Key | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `#FFFFFF` (pure white) | `colorScheme.surface` / `colorScheme.background` | Screen backgrounds; `Scaffold` background; `Card` default surface |
| Secondary (30%) | `#F5F5F5` (neutral grey-50) | `colorScheme.surfaceVariant` | Bottom navigation bar fill; section dividers; `ListTile` background on contact/campaign rows; input fill in `InputDecoration` |
| Accent (10%) | `#1565C0` (Material Blue 800) | `colorScheme.primary` | Reserved elements listed below |
| Destructive | `#C62828` (Material Red 800) | `colorScheme.error` | Destructive actions only (see list below) |
| On-surface text | `#212121` (Grey 900) | `colorScheme.onSurface` | All body text on white/grey surfaces |
| On-accent text | `#FFFFFF` | `colorScheme.onPrimary` | Text/icons on accent-colored buttons and surfaces |

**Seed color for `ColorScheme.fromSeed`:** `Color(0xFF1565C0)` (Blue 800). This generates the full M3 tonal palette automatically — use generated `primaryContainer`, `secondaryContainer`, `tertiary` for notification badges and status chips only; do not introduce additional brand colors.

**Accent (`#1565C0`) is reserved exclusively for:**
- Primary action buttons (`ElevatedButton` / `FilledButton`): "Send Now", "Create Account", "Buy Bundle", "Add Contact"
- Active state of bottom navigation bar icons and labels
- Hyperlinks and in-line text actions (e.g., "Forgot password?")

- Selected/focused state of form fields (cursor, underline, outline on focus)

**Destructive (`#C62828`) is reserved exclusively for:**
- Delete contact action (trailing icon in `ListTile` + confirmation dialog confirm button)
- "Cancel" button for a scheduled campaign that has been queued (not available at MVP but reserved)
- Error states: inline form validation errors, error banners, EXPIRED payment state indicator text

## Interaction Contracts

### Loading States

Every async operation follows one of two patterns. Never show an infinite spinner with no timeout.

**Write operations (online-only — D-05):**
1. User taps CTA button.
2. Button shows `CircularProgressIndicator` in place of label (`isLoading` state on button).
3. `LoadingOverlay` blocks screen interaction.
4. On success: dismiss overlay, navigate or update state.
5. On failure: dismiss overlay, show `ErrorBanner` or inline error. Button returns to active state. User can retry.

**Read operations (cache-first — D-05):**
1. On first load: show `CircularProgressIndicator` centered in content area (max 3 seconds wait before error state).
2. If cache available: immediately display cached data + `StaleIndicator`.
3. Live data arrives: replace/update content silently (no loading flash if cache was shown).
4. If no cache and network fails: show `errorNetworkLoad` error with `RefreshIndicator`.

### Pull-to-Refresh

Available on: Dashboard, Contact List, Campaign History, Notification Feed. Uses Flutter `RefreshIndicator` wrapping the `CustomScrollView`. On pull: re-fetch live data, update Hive cache, update `StaleIndicator` timestamp.

### NIDA Pending Auto-Poll

- Poll interval: 10 seconds (via `Timer.periodic` in `PendingPollerNotifier`).
- Poll mechanism: `POST /auth/refresh` — reads `status` from response body (not JWT claim decode — see RESEARCH.md Pitfall 3).
- On `status == "VERIFIED"`: `AuthNotifier` transitions to `AuthState.verified(...)` → go_router `refreshListenable` fires → redirect to `/dashboard` → show `verifiedSuccessMessage` snackbar.
- On timeout/error during poll: swallow silently, retry on next tick. Do NOT surface transient poll errors to the user.

### Payment Status Polling (STK Countdown)

- Countdown seeds from `timeoutSeconds` field in `POST /api/v1/payments` response (120 seconds).
- Poll payment status every 5 seconds via `GET /api/v1/payments/{id}`.
- On `status == "CONFIRMED"`: stop countdown, show success state (`stkSuccessTitle`), refresh wallet balance.
- On `status == "EXPIRED"` OR countdown reaches 0: stop timer, show expired state (`stkExpiredTitle`). Do NOT auto-retry.
- User taps "Try Again" → navigate to `/bundles` (catalog).

### Notification Feed Polling

- Poll interval: 30 seconds (per REQUIREMENTS.md "30-second polling is sufficient").
- Fetch `GET /api/v1/notifications?page=0&size=20` on each tick.
- Unread badge count = `items.where((n) => !n.read).length` — computed client-side.
- Badge disappears when user navigates to `/notifications` screen (mark-read is client-side derived; no PATCH endpoint required at MVP).

### Character Counter (Campaign Composer)

- Real-time as user types — no debounce needed (pure synchronous Dart computation).
- GSM-7 threshold: 160 chars per part (153 for multi-part).
- UCS-2 threshold (triggered by any non-GSM-7 character): 70 chars per part (67 for multi-part).
- Display format: `"120/160 · 1 SMS"` (GSM-7) or `"⚠ 45/70 · 1 SMS (UCS-2)"` (UCS-2).
- UCS-2 warning text uses `colorScheme.error` (`#C62828`) but does NOT block sending.
- Part count shown when >1 part: `"240/306 · 2 SMS (UCS-2)"`.

### Offline Write Attempt

- User taps any write CTA while offline.
- Detect via Dio `DioException.type == DioExceptionType.connectionError` (or `socketException`).
- Show `ErrorBanner` with `errorNetworkWrite` copy + "Try Again" button.
- Do NOT queue the write silently (D-05 — online-only writes, no offline queue).

### go_router Navigation Guards

Three redirect rules (evaluated in order on every navigation event and auth state change):

1. `AuthState.unauthenticated` + route not in `[/login, /register, /onboarding, /splash]` → redirect to `/login`.
2. `AuthState.pending` + route not `/pending` → redirect to `/pending`.
3. `AuthState.verified` + route in `[/login, /register, /onboarding]` → redirect to `/dashboard`.

The `GoRouter` must set `refreshListenable: GoRouterRefreshStream(ref.watch(authNotifierProvider.stream))` so the guard re-evaluates when auth state changes asynchronously (NIDA poll, session expiry).

## l10n Architecture

All user-facing strings must use `AppLocalizations.of(context)!` — never hardcoded literals in widget `build()` methods. This is a structural requirement (D-04), not a style guideline.

| Property | Value |
|----------|-------|
| Source locale | `en` (English) |
| Second locale | `sw` (Swahili — ISO 639-1 code) |
| ARB template | `lib/l10n/app_en.arb` |

| Generated file | `app_localizations.dart` (via `flutter gen-l10n`) |

| In-app toggle | `SegmentedButton<Locale>` with `[Locale('en'), Locale('sw')]`; persisted to `shared_preferences` under key `'selected_locale'` |
| Device-locale fallback | Default to device locale; if device locale not `en` or `sw`, fall back to `en` |

**ARB key naming rules:**
- Prefix by feature: `register*`, `login*`, `pending*`, `dashboard*`, `bundle*`, `stk*`, `contact*`, `composer*`, `campaign*`, `notification*`, `error*`
- Use `lowerCamelCase`
- Parameterized strings use `{paramName}` placeholder (e.g., `"Futa {name}?"`)

- "Already have an account? Log in" link below button.

### NIDA Pending (`/pending`)

- No bottom navigation. No back button (locked route).
- Full-screen centered layout (centered in `Column` with `MainAxisAlignment.center`).
- Top: animated pulsing circle (custom `AnimatedContainer` with scale animation) in `primaryContainer` color — 80 dp diameter.
- `titleMedium` heading: `pendingScreenTitle`.
- `bodyMedium` body (center-aligned, 80% screen width max): `pendingScreenBody`.
- `bodySmall` status row with `CircularProgressIndicator` (size 14 dp) + `pendingStatusLabel`.
- Bottom: `pendingLogoutLink` as `TextButton` with `colorScheme.error` foreground.
- **Auto-polls every 10 seconds** (PendingPollerNotifier) — no user action needed.

### Login (`/login`)

- `SingleChildScrollView` with 16 dp horizontal padding.
- App logo (48 dp) centered at top.
- Fields: Email, Password.
- `loginSubmitButton` (`FilledButton`, full-width).
- "Forgot password?" `TextButton` below (in-app navigation to password reset flow — email submission form).
- "Don't have an account? Register" link at bottom.
- On error: `errorLoginFailed` inline below Password field; no snackbar.

### Dashboard (`/dashboard`)

- `CustomScrollView` with `SliverAppBar` (pinned, expandable).
- `SliverAppBar` expanded content: `BalanceCard` — large `headlineLarge` credit count; `labelMedium` "SMS Credits" label; expiry warning row in `bodySmall` amber text if credits expire within 7 days.
- App bar actions: notification bell icon with `NotificationBadge`; language toggle icon.
- Below `SliverAppBar`: `SliverList` with:
  - Section header "Recent Campaigns" (`titleMedium`).
  - Up to 5 `CampaignListTile` items (most recent first).
  - "View all campaigns" `TextButton` link if more than 5 campaigns exist.
  - If no campaigns: `campaignsEmptyHeading` empty state.
- `FloatingActionButton.extended` (bottom right): `dashboardQuickSendFab` → navigates to `/campaigns/new`.
- `StaleIndicator` shown below balance card when displaying cached data.

### Bundle Catalog (`/bundles`)

- `AppBar` title: "Buy SMS Credits" / "Nunua Mikopo ya SMS".
- `ListView` of `BundleCard` items (one per active bundle).
- Bundles in ascending price order. Recommended bundle (Growth or closest mid-tier) highlighted with `colorScheme.primaryContainer` background on the card.
- Each `BundleCard`: bundle name (`titleMedium`) + SMS count (`bodyMedium`) + price in TZS (`headlineLarge`) + `bundleBuyButton` (`FilledButton`).
- Tapping "Buy Bundle" → opens bottom sheet asking for mobile money number + provider selection (`SegmentedButton` or `DropdownButton` for M-Pesa / Tigo Pesa / Airtel Money / Halo Pesa / AzamPesa) → confirm → navigate to `/bundles/purchase`.

### STK Purchase (`/bundles/purchase`)

- No bottom navigation. Back arrow disabled during active countdown.
- Centered layout: provider logo (or generic phone icon) + `stkInstruction` body.
- `CountdownWidget`: `CircularProgressIndicator` (large, 80 dp, `colorScheme.primary` color, seeds from `timeoutSeconds`) + `headlineLarge` mm:ss text center.
- `labelMedium` `stkCountdownLabel` above the ring.
- `bodyMedium` `stkWaitingBody` below the ring.
- **EXPIRED state:** replace countdown with `Icon(Icons.timer_off, color: colorScheme.error, size: 64)` + `stkExpiredTitle` (`titleMedium`) + `stkExpiredBody` (`bodyMedium`) + `stkTryAgainButton` (`FilledButton`).
- **SUCCESS state:** replace countdown with `Icon(Icons.check_circle, color: Colors.green.shade800, size: 64)` + `stkSuccessTitle` + `stkSuccessBody` + "Go to Dashboard" button.

### Contact List (`/contacts`)

- `CustomScrollView` with `SliverAppBar` (pinned) containing `SearchBar` in `bottom` slot.
- `SliverList` of `ContactListTile` items.
- Empty state: `contactsEmptyHeading` centered with `xl` top padding and "person_add" icon.
- `FloatingActionButton` (bottom right): `Icons.person_add` icon → `/contacts/add`.
- Long-press on `ContactListTile` → selection mode with delete action in `AppBar` (replaces standard bar).
- Or: trailing delete `IconButton` with `colorScheme.error` foreground → `deleteContactTitle` confirmation dialog.
- `RefreshIndicator` wraps `CustomScrollView`.
- `StaleIndicator` shown at top of list when offline cache is displayed.

### Add Contact (`/contacts/add`)

- `AppBar` title: "Add Contact" / "Ongeza Mawasiliano".
- Fields: Full Name, Phone Number (hint: `07xx xxx xxx`; auto-normalizes to E.164 on save).
- `addContactSaveButton` (`FilledButton`, full-width). Online-only write — shows `LoadingOverlay` during POST.
- On success: pop to contact list; updated list refreshes from API.
- On error: `ErrorBanner` with `errorNetworkWrite` + "Try Again".

### Campaign Composer (`/campaigns/new`)

- `AppBar` title: "New Campaign" / "Kampeni Mpya".
- Recipient section: "Send to" label + `TextButton` "Select contacts ({n} selected)" → opens contact picker modal (bottom sheet with multi-select `CheckboxListTile` items from Hive-cached contact list). At least 1 contact required before send.
- Message section: `TextFormField` with `maxLines: null` (expands with content), `minLines: 5`, no character limit enforced at field level (limit is visual, not hard).
- `SmsCharCounter` widget shown below the text field — updates on every keystroke.
- UCS-2 warning shown inline below counter in `colorScheme.error` text when non-GSM chars detected.
- `composerSendButton` (`FilledButton`, full-width) at bottom. Disabled until ≥1 contact selected AND message not empty.
- On send:
  - `LoadingOverlay` shown during `POST /api/v1/campaigns` + `POST /api/v1/campaigns/{id}/send`.
  - On success: navigate to campaign detail screen for the new campaign.
  - On `errorInsufficientCredits`: dismiss overlay, show `ErrorBanner` with "Buy Credits" action button.
  - On network error: `ErrorBanner` with "Try Again".

### Campaign History (`/campaigns`)

- `CustomScrollView` with `SliverAppBar`.
- `SliverList` with paginated `CampaignListTile` items.
- Infinite scroll pagination: load next page when user scrolls to within 200 dp of bottom.
- Empty state: `campaignsEmptyHeading`.
- `RefreshIndicator`.

### Campaign Detail (`/campaigns/:id`)

- `AppBar` title: campaign name (truncated with ellipsis).
- Summary card: total recipients, delivered count, failed count — displayed as three horizontal stat tiles.
- `CampaignStatusChip` for overall campaign status.
- Section header: "Message Delivery Status" (`titleMedium`).
- `ListView` of per-message rows: phone number + `CampaignStatusChip`. Paginated.
- Failed message rows have a subtle red-50 row background tint (not full `colorScheme.error`).

### Notification Feed (`/notifications`)

- `AppBar` title: "Notifications" / "Arifa".
- `ListView` of notification `ListTile` items: icon (varies by notification type) + message text + timestamp in `bodySmall`.
- Unread items: `ListTile` with `tileColor: colorScheme.primaryContainer` background.
- Read items: white/surface background.
- On screen open: mark all visible items as read (client-side only; `read` flag set locally until next poll overwrites).
- Empty state: `notificationsEmptyHeading`.
- Notification type → icon mapping:
  - `NIDA_VERIFIED` → `Icons.verified_user` (green)
  - `PAYMENT_CONFIRMED` → `Icons.payment` (blue)
  - `LOW_CREDIT` → `Icons.warning` (amber)
  - `CREDIT_EXPIRY` → `Icons.timer` (amber)
  - `CAMPAIGN_COMPLETED` → `Icons.send` (blue)
  - `SENDER_ID_DECISION` → `Icons.badge` (blue or red depending on approved/rejected)
