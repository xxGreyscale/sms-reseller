Close backend gap D-13 (and optional D-14): add GET /auth/me to identity-service so the Flutter NIDA
PENDING screen can poll verification_status cheaply WITHOUT calling /auth/refresh (which rotates the
7-day refresh token on every call — ~60k rotations over a poll window). Optionally add
PATCH /api/v1/notifications/{id}/read for the in-app feed badge (the Notification.read field already
exists). D-14 is nice-to-have; include it in this plan since it is a thin, same-pattern addition and
the notification-service is otherwise untouched this phase.

Purpose: 06-07 (NIDA pending poll) calls GET /auth/me on a 10s timer; 06-12 (notification feed) may
use mark-read. (NOTE: UI-SPEC § NIDA Pending Auto-Poll currently specifies POST /auth/refresh; with
/auth/me available the executor of 06-07 should poll /auth/me instead per CONTEXT D-13 — the cheaper,
non-rotating path. The badge in 06-12 is derived client-side, so D-14 remains optional for the app.)
Output: GET /auth/me + MeResponse + IT (identity); PATCH read endpoint + service + repo + IT (notification).
</objective>
