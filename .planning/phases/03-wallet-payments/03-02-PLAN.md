<objective>
The lot-based append-only ledger core: CreditLot + CreditTransaction entities and migrations, derived BalanceService, and the expiry-soonest-first ReservationService using `SELECT FOR UPDATE` (D-01, D-02). This is the financial-correctness foundation every other wallet feature depends on, so it is built TDD-first.

Purpose: Guarantee no negative balance, no double-spend, and correct expiry exclusion before any consumer or API touches the ledger.
Output: A locked, tested ledger with grant/reserve/balance primitives and two Flyway migrations.
</objective>
