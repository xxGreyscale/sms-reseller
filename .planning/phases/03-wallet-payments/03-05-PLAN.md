<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Azampay webhook → public callback endpoint | Unauthenticated inbound; spoofable; replayable |
| concurrent/duplicate callbacks → payments table | Double-credit boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Tampering | Duplicate callback double-emits PaymentConfirmed | mitigate | SUCCESS idempotent guard + outbox event_id UNIQUE; CallbackProcessingIT asserts single outbox row on duplicate |
| T-03-12 | Spoofing | Forged success callback | mitigate | WebhookSignatureValidator (StubSignatureValidator dev; HMAC deferred to onboarding) + utilityRef must exist in payments (404 otherwise) |
| T-03-13 | Availability | Infinite PENDING / no terminal state | mitigate | PaymentTimeoutJob marks EXPIRED at 120s (PYMT-07); status endpoint always surfaces a terminal-or-EXPIRED state |
| T-03-14 | Tampering | Lost PaymentConfirmed on crash after status flip | mitigate | Outbox row written in same TX as SUCCESS flip (transactional outbox); relay publishes at-least-once |
| T-03-15 | Tampering | Two concurrent PENDING payments per user | mitigate | existsByUserIdAndStatus check + uq_payments_user_pending partial unique index (DataIntegrityViolation→409) |
</threat_model>
