<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| identity.events AMQP → wallet consumer | At-least-once delivery; replay possible |
| client→wallet read API | JWT-validated; user must only see own data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Tampering | Duplicate UserVerified delivery double-grants bonus | mitigate | processed_events ON CONFLICT DO NOTHING; UserVerifiedConsumerIT proves single grant on re-delivery |
| T-03-09 | Elevation of Privilege | IDOR — user reads another user's balance/history | mitigate | userId sourced only from JWT subject; ITs assert cross-user exclusion (ASVS V4) |
| T-03-10 | Repudiation | Lost bonus grant on consumer crash mid-process | mitigate | @Transactional consumer — processed_events insert + lot grant commit together or roll back together |
</threat_model>
