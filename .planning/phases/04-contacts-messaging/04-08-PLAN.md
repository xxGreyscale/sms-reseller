<objective>
Add scheduled campaign dispatch with cancellation, and the sender-ID request/approval backend.
The DB-polled @Scheduled job dispatches SCHEDULED campaigns at/after their time (reusing the
04-05 executeSend pipeline); cancellation flips state so the poller skips it. Users submit
custom alphanumeric sender-ID requests; an admin-only internal API approves/rejects them via the
REQUESTED→APPROVED|REJECTED state machine, emitting a SenderIdDecided outbox event for Phase 5.
Delivers MESG-04 (schedule), MESG-05 (cancel), SNDR-02 (request), SNDR-03 (admin decide
backend), SNDR-04 (decision notification event).

Purpose: Completes the remaining Phase 4 backend surface; admin panel UI is Phase 5.
Output: Scheduler + cancel + sender-ID lifecycle with decision events.
</objective>
