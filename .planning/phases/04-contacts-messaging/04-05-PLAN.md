<objective>
Build the send-campaign request path: expand recipients from target groups via contact-service,
exclude suppressed numbers, reserve credits synchronously from wallet (Resilience4j-wrapped,
campaignId as idempotency key) and transition the campaign to QUEUED only on success, snapshot
recipients into outbound_messages with a per-message lotId (using the per-lot allocation from
04-07), publish one AMQP message per recipient to the quorum send queue, and consume those
messages calling SmsProvider — emitting MessageAccepted (CONSUME) on accept and MessageReleased
(RELEASE) for slots never handed to the carrier. Delivers MESG-03 (reserve-before-QUEUED +
insufficient error), MESG-08 (post-send confirmation), MESG-09 (suppression exclusion).

Purpose: This is the spine of the platform — verified, credit-backed bulk dispatch.
Output: Working immediate-send pipeline with credit reservation and per-message lot correlation.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | Tampering/Spoofing | campaign send without credits | mitigate | QUEUED transition only after successful reserve(); 409 → InsufficientCreditsException, campaign not queued (MESG-03). |
| T-04-11 | Tampering | wrong lot charged | mitigate | per-message lotId from per-lot allocation (04-07) carried in payload + outbound_messages; CONSUME targets that lot (D-13, Pitfall 4). |
| T-04-12 | Information disclosure | IDOR on campaign send | mitigate | userId from JWT subject; recipients expanded only for that user's groups; 404 on cross-user campaign id. |
| T-04-13 | Coupling/Distributed-txn | sync HTTP in AMQP consumer | mitigate | consumer emits events only; reserve is the sole sync call and it is on the request path (D-03/D-04). |
